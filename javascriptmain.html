JavaScript

JavaScript acts as the body’s muscle, bringing functionality to a website. Makes websites more dynamic. An actual programming language.

Data Types

A data type is a specific kind of data/information, different types will be interpreted differently by the program.

Strings - primitive JavaScript data type. A series of characters enclosed in either double quotes or single quotes. (can hold any character)

Numbers - primitive JavaScript data type. Don't need quotes.

Printing in the console typically has this syntax:

console.log();

With your string or variable in the parentheses. JavaScript will still work if there wasn’t a semicolon at the end, however it’s good practice especially if you plan to learn other languages.

Variables

How to declare and initialize a variable.

var defines a variable, followed by an arbitrary name to refer to the variable. Follow camelCase rules. Only the first letter of a second/third/etc. word in a variable name should be capitalized. The first word is completely lowercase. = or the assignment operator will determine what value the variable holds. In this case the value is a string of my name. Any variable can be printed to the console.

If you ever need to reassign a variable, just type the name of the variable you want to change followed by the assignment operator then the new value you want.

Math with Variables

Numbers can also be stored in variables. Variables that store numbers can be reassigned with a math operation. Math operators can be applied to numbers like usual, parenthesis, and order of operations all act regularly in JavaScript. Also math can be applied to variables that hold numbers.

Interpolation

Interpolation is including variable data in sentences we want printed out by the console.

Important note: interpolation doesn’t use quotes. It uses the back tick at the top left of the keyboard. If you use quotation marks the interpolation won’t work.

Functions

A function is basically an action in our code. There are a lot of functions when making a website. For example, clicking a button and something happens, like new data appearing on the screen. These are a result of functions in the code. Know that a function will only occur when a specific action is performed, but can occur as many times as we need. Before we can make a function happen we need to know how to declare or write a function.

Whenever you’re declaring or creating a function, always start with “function” followed by the name you would like the function to be. Typically this name describes what the action is and should follow camelCase. Followed by the name is open and closed parentheses then open and close curly braces. Whatever code you put in the curly braces will be what happens when you call the function. To call the function so it does something just type the function name with the parenthesis.

Functions with Parameters

In order to make functions more specific rather than general we can make use of arguments and parameters. Parameters are what’s written in the parenthesis and can be written when you’re actually declaring your function. They act like a variable. So, when you write your function with a parameter variable make sure you follow the rules for interpolation. You can also have multiple parameters as long as they’re separated by a comma and space. When you call the function, the parenthesis before were empty, but now you can put in a value or argument to assign to your parameter. Although we’re looking at a string here, if you use a number parameter you can use it like a variable to do calculations in your function.

Functions that Return

Sometimes you might not want to directly print out the output in the console, but want to use the value for something else. In this case we can use return values. However there are important things to note when using return. A function can only return one value, make sure to explicitly state what you want to return, and the return line should be the last one in your function. The program will exit the function once the return line is read, which means any code after the return within the function will be ignored.

Arrays

Arrays are a complex data type unlike strings or numbers. An array is an ordered collection that stores multiple values, and can store any kind of element(strings, numbers, other arrays) as long as they are all the same type. In JavaScript, an element is each item inside of an array. An array will start and end with these brackets[ ] with each element separated by a comma. Most of the time they are assigned to variables.

Every element in an array is automatically assigned a number called an index starting from 0 and on. We can use the index to access a specific element of the array. Syntax for this is the name followed by the index in square brackets.

We can also see how many elements are in an array by using the .length property.

You can use the index access to update the array or an element. Use the syntax to access the index then the assignment operator followed by what you want to update it to.

If you don’t want to replace an element there’s also the ability to add or remove elements. To add an element use the .push() method, which will add the element to the end of the array. The inside of the parenthesis should be what you want to add.

If you want to remove an element you can use either the .pop() or .shift() method, which will remove the last and first element respectively. Neither need an argument unlike the .push() method.

Iteration

Iteration is the process of repeating a task for each item in a collection. Iteration is more efficient for us since we don’t have to write a function for each specific element in an array. We can just write one function and have it repeat for each element.

languages.forEach(function(name) { console.log(name); });

.forEach() is a built in function that runs through every element in an array. The only argument it can take is a function. “name” is the variable representing the current element being iterated over. “console.log” is the code that will run for each element. To take iteration even further we can learn how to use placeholder arrays. This is because we may not want to do the same thing to each element.

Objects

An object is a complex data type and a collection like an array, however each item has a label rather than an index number. Each data value has a label or key. key/value pairs provide an associative relationship. To further explain, an associative relationship means having a title/label for the info. This is different from arrays, which is one reason we might use it, and the other is that we don’t need it to be in a specific order.

Object Syntax

Start with the syntax for a regular variable. Unlike arrays, objects use the curly braces. Keys are on the left followed by a colon and then their associated value. Keys should follow camelCase. In this case languages are the key while the value is the country they’re spoken in. Each pair should be separated by a comma. Another difference from arrays is that objects can hold multiple data types like strings and numbers while arrays can only hold one type. In order to access a value in an object we use dot notation syntax. It should be the name of the object then a dot then the label or key of the value we want. To make things even more complex, know that you could also make an array of objects.

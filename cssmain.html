CSS

CSS stands for Cascading Style Sheets. Acts as the body’s skin/design and style elements of the website. Also not a programming language. Can make the same HTML file look completely different.

Common Uses

  • Text: font, size, weight, and color can all be changed
  • Borders: has to be defined in CSS if you want it, not automatic, can apply to specific sides or all of them, can set the thickness, style, and color

Buttons

With buttons, css can get more advanced. button:hover is a pseudo-selector. This means that only when the button is hovered over will this rule be applied. All other times it will be ignored. cursor: pointer; will change what the cursor looks like when it hovers over the button. An additional property that can be applied is border-radius. This will round the corners of the border based on the value you put in.

Layout

As we know, HTML elements have a default layout behavior. With CSS we can override any element with the layout behavior we want. We can force button elements to have block behavior, or paragraph elements to be inline.

The Box Model

The Box Model is a model to help control layout and design in CSS. Each element can be considered a rectangle. Every element consists of each piece listed above. Content, the actual text is surrounded by padding which gives space between the content and the border. The margin creates space outside the border and between elements. By default most elements don’t have any padding. Padding is a CSS property, it can be applied all around or to specific sides. The same concept can be applied to margins. Keep in mind that padding is within the border while margins are outside the border.

Default Browser Behavior

Every time you create a new project, the browser will put in some default margin and padding. This is mainly to help developers who don’t plan to use CSS. Since we do, it can affect our layout. To truly design and customize our sites we should use this code in CSS to “reset” those properties.

Sizing Images

I previously mentioned the width and height of an element. This can be easily explained through sizing images. A lot of the time images we put in won’t be the size that we want. We can fix this by using the width and height properties in CSS. Typically, you only want to use either width or height since using both will change the ratio of the image. If you only use one of the two the other will naturally change based on the original ratio. If you do want to change both, then the object-fit property may be helpful. If you want to use both but the ratio of the image changes and it becomes too squished or stretched, then the object-fit property will solve this for you. This declaration

object-fit: cover;

Will alter the photo so it will fit the new width and height without looking odd. Although the photo now matches the desired quality, some of the original image may be cut out from the resize. See the difference below.

Flexbox

Before learning the actual properties it’s important that you understand nesting first. You should know that any element that wraps other elements is considered a parent/container. Any elements that are wrapped by another element are considered a child/item. Any element that’s directly under a parent (1 indentation) is considered a direct child. If another element is wrapped by the direct child (2 indentations) then it is not a direct child of the initial parent but technically a grandchild while a direct child of the middle element. Always add the

display: flex;

declaration to the parent element. Also, when using flexbox properties, the parent itself doesn’t change, but the way its children elements are organized/look may change.

Flexbox Properties

  • justify-content: center children, space around/between children, control how content sits on main axis(default=horizontal)
  • align-items: control how content sits on secondary axis
  • flex-direction: changes direction of the flex container; direction of container defines main axis
  • flex-wrap: by default flex box tries to keep all children on one line(horizontal or vertical), this property allows children to wrap onto the next line(s)

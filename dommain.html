The DOM

(Before getting into the DOM, this will only be a brief introduction due to the overwhelming amount of content regarding the DOM. There are plenty of outside resources that provide more information on this topic.)The DOM aka Document Object Model represents how the browser reads HTML. It allows JS interaction w/ HTML allowing us to manipulate, structure, and style websites. Going along with our body analogy, the DOM acts as the body’s brain. DOM manipulation is making changes in the browser that are prompted yet not directly created by the user. Now that we’re learning about the DOM, we can learn about jQuery. JQuery is a popular library that has chunks of code in JavaScript that someone else wrote. This means that there’s less for us to write. Before using any code from the library make sure there's a script element leading to JQuery within the head element of your HTML. Some IDEs, like Replit, will already have this in there.

Access HTML Elements

First of all we do a lot of work in the console through our IDE. However you could also view the console for any webpage on chrome. Here’s how to access chrome developer tools. Right click the page then Inspect. A popup on the right side should appear with stuff like elements and console. Console works just like the one we use in Replit, so it isn’t totally necessary sometimes. However, chrome developer tools don’t open/ work on school chromebooks. Before we make things functional we have to know how to access elements from the DOM first. Luckily, jQuery has built-in features that allow us to easily access elements from the DOM.

The syntax in JavaScript is similar to what you would use for interpolation. However, in this situation it has a different purpose and meaning. “$” is what tells the browser to go to HTML and find the argument/element. Inside the parentheses is the actual element/argument you want it to find. Instead of the type of element you could also access elements by their class name for more specificity. Since we stored this in a variable we can console.log the variable to see the element. The last line should be what prints out in the console, which should be the text in that element in HTML.

Events

An event is basically any action that a user takes while on our site eg. clicking, scrolling, hovering, etc. Events are what really allow DOM manipulation to take place. First, you need an event listener. An event listener will sit and wait for an event to take place then call a function when it does. Then, there are event handlers. Event handlers are the actual function that’s called when an event occurs.

Access CSS

Similar to HTML where we can change the text that appears on the screen, with CSS we can change the style of the text when an event occurs.

First we declare variables and store a button and div in them. Next is declaring the event listener. Then, the function changeBackgroundColor() is our event handler. Inside the function the container variable is called to go to CSS rules of the div element through “.css” which takes 2 arguments: a property and a value. “backgroundColor” is the property then comes the value you want to update the property to.
